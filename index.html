2. Native与Flutter通信

2.1. 概述

阅读前建议先跟着官方教程过一遍demo，以获得基本概念。

如图所示,当在Native端初始化一个Channel并注册处理消息的Handler时，会同步生成一个与之对应的BinaryMessageHandler。BinaryMessageHandler以channel name 为key注册到BinaryMessenger中。

当Flutter端发送消息到BinaryMessenger时，它以入参channel找到对应的BinaryMessageHandler，交由其处理。Channel从BinaryMessageHandler接收到二进制消息后，通过Codec解码后传递给Handler处理。

Handler处理完毕后，回调result。并将result通过Codex编码为二进制数据格式，通过BinaryMessenger发送回Flutter端。

2.2. 概念详解

2.2.1 Channel概述

Flutter共定义了3种不同类型的Channel，分别是

  • BasicMessageChannel:用于传递字符串和半结构化的信息,持续通信,收到消息后可以回复此次消息。比如Flutter将从服务端陆续获取到的信息交给Native加工,Native处理完返回等。
  • MethodChannel: 用于传递方法调用(method invocation)。一次性通信,如Flutter调用Native拍照;
  • EventChannel: 用于数据流(event streams)的通信。持续通信,收到消息后无法回复此次消息,通常用于Native向Dart的通信,如:网络连接变化、传感器等;

每种Channel均有三个重要成员变量:

  • name:String 类型,代表Channel的名字，也是其唯一标识符;
  • binaryMessenger:BinaryMessenger类型,代表消息信使,是消息的发送与接收的工具;
  • codec:MessageCodec或MethodCodec类型，代表消息的编解码器;

2.2.2 channel name

一个Flutter应用中可能存在多个Channel,每个Channel在创建时必须指定一个独一无二的name,Channel之间使用name来区分彼此。当有消息从Flutter端发送到Native端时,会根据其传递过来的channel name 找到该channel对应的Handler。

2.2.3 消息信使:BinaryMessenger类型

BinaryMessenger是Native端与Flutter端通信的工具,其通信使用的消息格式为二进制格式数据。当我们向channel注册处理消息的Handler时，会以channelName为key,存储在BinaryMessenger中。

当Flutter端发送消息到BinaryMessenger时，会根据name找到对应的Handler来处理。

从源码中可以看到,接收到的是被编码的二进制数据,需要通过codec解码后，才能作为handler的入参。当handler处理完毕后,codec又会重新编码为二进制数据返回。

BinaryMessenger在Android端是一个接口，其具体实现为FlutterNativeView。在iOS端是一个协议,名为FlutterBinaryMessenger,FlutterViewController遵守该协议。

2.2.4 消息编解码器 Codec

Codec用于将二进制格式的数据转化为Handler能够识别的数据,Flutter定义了两种Codec:

  • MessageCodec:二进制数据与基础数据之间的编解码。BasicMessageChannel所用。
  • MethodCodec:二进制数据与MethodCall对象和返回结果之间的编解码。MethodChannel和EventChannel所用。

MessageCodec有以下实现:

  • BinaryCodec

    此种方式没有做任何操作，只是将二进制数据消息返回。

  • StringCodec

    此种方式用于字符串与二进制数据之间的编解码，格式为UTF-8

  • JSONMessageCodec

    此种方式用于基础数据与二进制之间的编解码，支持基础数据、列表、字典。

  • StandardMessageCodec

    此种方式是BasicMessageChannel的默认编解码器，支持基础数据、字典和列表。

MethodCodec有以下实现:

  • JSONMethodCodec

    此种方式的编解码依赖于JSONMessageCodec,当在编码MethodCall时,会先将MethodCall转化为字典{“method”:method,”args”:args}。当在编码调用结果时,会将其转化为一个数组,调用成功为[result],调用失败为[code,message,detail]。再使用JSONMessageCodec将字典或数组转化为二进制数据。

  • StandardMethodCodec

    MethodCodec的默认实现，同上,依赖于StandardMethodCodec.

Flutter系列-基础概念

2019-07-19

Widget

Widget本身不是视图，并且不直接绘制任何东西，而是对UI及其语义的描述，可以认为它是”声明和构建UI的方法”。因而Widget是轻量级的，重新构建的代价很小。

在Android和iOS中，我们可以拿到对应的视图进行直接操作更新。在Flutter中,Widget是不可变的,我们需要通过操作Widget的状态来更新,这就是StatefulWidget和StatelessWidget的来源。两者的区别在于StatefulWidget有一个State对象来存储状态数据并传递到树重建中。

异步编程

Dart是单线程模型，没有主线程/子线程之分。Dart也是Event-Looper以及Event-Queue的模型，所有的事件都通过EventLooper依次执行。其中存在两个EventQueue:Event Queue和microtask Queue。Event-Looper优先全部执行完Microtask Queue中的Event才会执行Event Queue中的Event。

通过dart:async库中的Api可以进行异步任务调度。但要尽量避免Microtask Queue过于庞大，避免引起阻塞。 

- 使用Future类，可以将任务加入到Event Queue的队尾;
- 使用scheduleMicrotask函数，可以将任务加入到Microtask Queue队尾;

然而有时需要处理大量的数据，这会导致UI挂起。此时需要使用Isolate来发挥多核心CPU的优势来处理那些长期运行或计算密集型的任务。

Isolate是分离的运行线程，并且不和主线程的内存堆共享内存。这意味着不能访问主线程中的变量或者使用setState来更新UI。

Go语言圣经04-函数

2019-07-12

Go语言的函数调用栈为可变栈，栈的大小按需增加。使得我们在使用递归等场景时不必考虑溢出和安全问题。

Go语言中函数是first-class,像其他值一样拥有类型，可以被赋值给其他变量，传递给函数，从函数返回。函数类型的零值是nil，调用nil的函数会引起panic错误。

defer语句经常被用于处理成对的操作，如打开、关闭、连接、断开连接、加锁、释放锁。通过defer机制，不论函数逻辑多复杂，都能保证在任何执行路径下，资源被释放。多条defer语句遵循先进后出的栈顺序执行。

Go语言中可以给任意类型定义方法，因此我们可以方便的为一些简单的数值、字符串、切片等附加行为。

在包外调用时，方法会比函数调用更简洁方便。函数调用必须写全包名，而方法可以通过实例变量直接调用。

不管方法的接收者是指针类型还是非指针类型，都是可以通过指针/非指针类型进行调用的，编译器会帮你做类型转换。需要关注的是，如果声明为非指针变量，调用会产生一次拷贝;如果声明为指针类型,他指向的始终是一块内存地址，就算你对其进行了拷贝。

接口类型是对其他类型行为的抽象和概括。也就是说当拿到接口类型时，不知道它是什么，唯一知道的就是可以通过它的方法来做什么。

Go语言圣经03-数据类型

2019-07-11

Go语言将数据类型分为四类：基础类型、复合类型、引用类型和接口类型。

字符串

字符串的值是不可变，因此尝试修改字符串内部数据的操作也是被禁止的。不变性意味着如果两个字符串共享相同的底层数据的话也是安全的，这使得复制任何长度的字符串代价是低廉的。

数组和切片

数组的长度是固定的，因此在Go语言中很少直接使用数组。和数组对应的类型的Slice切片，它是可增长和收缩的动态序列。

数组的长度是数组类型的一个组成部分，因此[3]int和[4]int是两种不同的数组类型。

一个Slice由三个部分构成:指针、长度和容量。指针指向第一个slice元素对应的底层数组元素的地址。长度对应slice中元素的数目。容量是从slice的开始位置到底层数据的结尾位置，容量>=长度。

当使用append追加元素时，如果容量不够，内部会重新开辟新的数组，再进行元素的复制。此时切片底层引用了新的数组。

结构体

考虑效率，较大的结构体通常会用指针的方式作为参数传入和返回。而如果需要在函数内部修改结构体成员的话，必须用指针的方式，因为在Go语言中参数都是值拷贝传入的，函数参数不再是函数调用时的原始变量。

在结构体中只声明成员对应的数据类型而不指名成员的名字，这样的成员叫匿名成员。一般用于结构体中嵌入另一个结构体作为成员，而又想避免struct.member.property这样繁琐的调用。

Go语言圣经02-程序结构

2019-07-10

命名

Go语言中包括函数名、变量名、包名等所有的命名都遵守相同的命名规则:以字母或下划线开头，后跟任意数量的字母、数字或下划线，并且区分大小写，不能使用保留关键字。

变量和函数名的首字母是大写，表示可以被外部的包访问。如fmt包的Printf函数。

包本身的名字总是用小写字母。

声明

每个Go源文件总是以包的声明开始，package name 说明源文件属于哪个包。包声明语句之后是import语句导入依赖的其他包，然后是包一级的类型、变量、常量、函数的声明语句。

Go语言主要有四种类型的声明语句:var、const、type和func，分别对应常量、变量、类型和函数的声明。

变量

变量声明的一般语法如下:

var 变量名字 类型 = 表达式

其中”类型”或”=表达式”可以省略其中一个。如果省略类型信息，那么将根据表达式来推导;如果省略初始表达式，那么将用零值初始化。

数值类型的零值为0，布尔类型的零值是false,字符串类型的零值是””,引用类型的零值是nil。零值机制使得Go中不存在未初始化的变量。

可以使用 “名字 := 表达式”的简短形式声明变量，此时类型自动推导。仅适用于局部变量的声明。

表达式new(T)创建一个T类型的匿名变量，返回变量地址，即*T。且每次都是返回一个新的变量地址，因此下面两个地址是不同的。

包一级声明的变量，其生命周期和程序的运行周期的一致的。而局部变量在不再被引用之后，可能会被垃圾回收机制回收。

编译器会自动选择在栈上还是在堆上分配局部变量的存储空间，而并非由声明方式决定。如下面的x变量必须分配到堆上，因为在函数外还是能通过包一级的global变量找到。

包和文件

每个包都对应一个独立的命令空间。如image包中的Decode函数和unicode/utf16包中的Decode函数是不同且隔离的。要在外部引用，必须显示使用image.Decode或utf16.Decode.

包名一般是短小的名字,且和导入路径的最后一个字段相同。

每个包在解决依赖的前提下，以导入声明的顺序初始化，每个包只会被初始化一次。因此如果一个p包导入了q包，那么在p包初始化的时候可以认为q包必然已经初始化过了。main包最后被初始化。

每个文件可以通过init初始化函数来初始化工作，在程序开始时按照声明的顺序被自动调用。初始化函数不能被调用或引用。

理解环境变量

GOROOT:

是Go语言安装根目录的路径，也就是上文中的/usr/local/go。如果一台机器上有多个Go版本并存，就需要注意GOROOT的设置来切换Go版本。

GOPATH:

是Go语言的工作目录,决定了使用Go命令如go install、go get 要把执行文件和源码放到哪个位置，决定项目中的Go依赖包是如何导入的。目录里分别有bin、pkg、src3个文件夹。

bin内存放可执行文件,go install 命令会把可执行文件放入该目录。

pkg存放编译打包后的lib，主要的*.a文件。

src内存放项目源码文件，包括go get获取的第三方依赖包。需要注意的是，GOPATH是整个Go工作环境，不是单个项目的环境(当然你也可以通过设置一个项目一个GOPATH)。具体的目录结构如下所示:

$GOPATH
    bin
    pkg
    src
        github.com
        项目A
        项目B
        ...

Go Module

GOPATH没有依赖包的版本控制，之后社区出现了各种各样的依赖管理方案。而Go Module则是官方的依赖管理方式。

表联结

内联结将一个表中的行与另一个表中的行想关联。

外联结还包含没有关联行的那些行，如用户表和订单表，没有下单的用户，在订单表中就没有对应用户id的行。使用外联结就能查出这些数据。使用 用户表 left outer join 订单表。
